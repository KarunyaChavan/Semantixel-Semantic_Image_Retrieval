# Security Policy

## Supported Versions

Security fixes are primarily provided for the latest version of Semantixel available on the `main` branch.

Users are encouraged to stay up to date with the latest commits and releases to receive security patches and dependency updates.

| Version        | Supported |
| -------------- | --------- |
| `main` branch  | ✅         |
| Older versions | ❌         |

---

## Reporting a Vulnerability

If you discover a security vulnerability in Semantixel, please avoid creating a public GitHub issue.

Instead, report the issue privately through one of the following methods:

1. **GitHub Security Advisories**
   If enabled for this repository, use the **Security → Advisories → Report a vulnerability** feature.

2. **Direct Contact**
   Contact the maintainer privately through GitHub.

---

## What to Include

Please include as much of the following information as possible:

* Description of the vulnerability
* Steps to reproduce the issue
* Potential impact
* Environment details (OS, Python version, dependencies)
* Suggested fix or mitigation (if available)

---

## Response Process

After receiving a report:

* The issue will be reviewed and validated.
* A fix will be developed if the report is confirmed.
* Security patches will be applied to the `main` branch.
* Public disclosure may occur after a fix is available.

Thank you for helping improve the security of Semantixel.
